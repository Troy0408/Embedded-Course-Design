# 嵌入式课程设计项目检查报告

检查日期：2026-07-06

项目名称：基于 STM32F103C8T6 的智能手表设计

## 1. 课程设计要求理解

根据项目根目录下 `嵌入式系统课程设计.md`，课程设计关注以下内容：

- 阶段材料：项目计划书、系统设计文档、中期检查报告、验收演示材料。
- 作品功能：基于 STM32F103C8T6 的智能手表，至少包含 OLED 时间显示、MPU6050 传感器数据读取和 FreeRTOS 多任务。
- 扩展功能：旋转编码器菜单切换、蓝牙数据同步和上位机、PCB 或计步功能。
- 文档要求：硬件方案、软件架构、FreeRTOS 任务划分、测试数据、功能展示、物料成本和个人总结报告。
- 验收要求：现场演示系统功能，并提交阶段性成果、演示视频或测试记录。

## 2. 项目当前完成情况

### 已满足部分

- 固件主工程 `SmartWatch_FreeRTOS/` 已形成完整 STM32 FreeRTOS 工程。
- Android 上位机 `SmartWatch_AndroidHost/` 已实现 HC-05 SPP 连接、状态显示和控制命令。
- OLED 已实现 5 个页面：表盘、IMU、步数、蓝牙、设备信息。
- 表盘页保留核心显示内容：状态栏、电量、BT/IMU 状态、大号时间、日期和页指示器。
- 旋转编码器可以切换页面，PB1 可以返回表盘。
- MPU6050 页面能够显示传感器数据和 I2C 检测状态。
- 蓝牙页面能够显示 HC-05 连接状态和帧计数。
- 手机 App 已实测可连接 HC-05，并能操纵手表功能。
- 已确认 HC-05 数据模式使用 `9600 8N1`，固件、OLED 文案和文档已同步。
- FreeRTOS 任务划分合理，包含传感器、显示、蓝牙、时钟等任务。
- 代码按驱动/应用模块拆分，结构清晰，主要模块职责明确。
- 根目录 README 已补充项目简介、硬件组成、软件架构、编译烧录和使用说明。
- 已添加根目录 `.gitignore`，便于上传 Git 仓库时排除生成物和本地缓存。

### 还需要完善的部分

- 课程报告、答辩 PPT、演示视频、功能截图仍需手动补充。
- 若课程要求提交中期检查报告，需要根据当前实测结果另行整理。
- 计步算法为基础版本，建议补充 100 步实测数据和误差说明。
- 未完成 PCB 设计；当前方案使用 STM32 最小系统板和模块化连线，应在报告中说明。
- Android APK 未在当前环境做最终打包，建议在 Android Studio 中完成一次 `assembleDebug` 或直接手机安装验证截图。
- 根目录原始课程要求文件存在编码显示异常，但原文件保留未改动。

## 4. 代码结构检查

### STM32 固件

- `Core/Src/app_freertos.c`：应用任务调度和共享数据维护。
- `Core/Src/smartwatch_ui.c`：OLED 页面绘制。
- `Core/Src/oled.c`：OLED 底层绘图。
- `Core/Src/mpu6050.c`：MPU6050 驱动。
- `Core/Src/encoder.c`：旋转编码器输入。
- `Core/Src/bluetooth.c`：蓝牙协议和串口收发。
- `Core/Src/step_counter.c`：计步算法。

结构总体清晰。部分调试用字段仍保留在数据结构中，用于页面外调试和后续扩展；本次只移除了主表盘页面上的调试显示，不影响其他页面和功能。

### Android 上位机

- `BtProtocol.java`：蓝牙帧构建与解析。
- `BluetoothSppClient.java`：后台 SPP 连接、读写和回调。
- `MainActivity.java`：单屏上位机 UI。
- `BtProtocolSmokeTest.java`：桌面协议烟测。

结构满足课程设计上位机要求。

## 8. 是否适合作为最终提交版本

当前项目适合作为课程设计最终代码提交版本。固件和 Android 上位机功能已经基本完整，核心演示链路已经打通，代码结构和文档也具备提交基础。

仍需注意：最终课程提交不应只提交代码，还应补充报告、PPT、演示视频/截图和测试记录。若老师强调 PCB，则需要说明本项目使用最小系统板和模块化连线完成，扩展要求中以蓝牙上位机和计步功能作为重点完成项。
